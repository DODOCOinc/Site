Introdução

O que cada componente faz?

Arduino Leonardo

É o ator da programação. Ele lê o botão e decide ligar ou desligar o LED.

LED

É a saída visual. Ele mostra a resposta do circuito acendendo ou apagando.

Resistor

Protege o LED, diminuindo a corrente elétrica para evitar que ele queime.

Botão

É um sensor de entrada. Quando pressionado, informa ao Arduino que uma ação aconteceu.

Protoboard

Ajuda a organizar as conexões sem solda, permitindo montar e testar o circuito.

Fios

Levam os sinais entre Arduino, botão, resistor e LED.

Prática ativa

Montar no Tinkercad

No Tinkercad, a dupla testa o circuito antes de usar os materiais físicos. O botão vira uma entrada: ele envia um sinal. O LED vira uma saída: ele mostra a resposta.

  • Adicionar Arduino, protoboard, LED, resistor e botão.
  • Ligar o LED a um pino digital, por exemplo D9.
  • Ligar o botão a outro pino digital, por exemplo D2.
  • Usar resistor no LED para proteção.
  • Usar resistor no botão para deixar o sinal estável.
  • Iniciar a simulação e testar o botão.
Fala para a turma:

O botão pergunta: “alguém me apertou?”. O Arduino Leonardo escuta essa pergunta e responde ligando ou desligando o LED.

Programação visual

Blocos para botão acionar LED

A lógica é simples: o Arduino Leonardo lê o botão. Se o botão estiver pressionado, o LED acende. Se não estiver pressionado, o LED apaga.

  • O botão é configurado como entrada.
  • O LED é configurado como saída.
  • O bloco “se” toma a decisão.
  • O LED acende quando a leitura do botão estiver ativa.
  • O LED apaga quando o botão não estiver pressionado.
Ponto-chave:

Entrada é o que o sistema percebe. Saída é o que o sistema faz depois de perceber.
